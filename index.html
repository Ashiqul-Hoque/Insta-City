Question Answer

Javascript(JS) is a single threaded and single concurrent interpreted programming language. It means this language reads and translates the file line by line without compiling a program into machine instructions.
To converts its language to machine code it needs an engine in the browser. Different browser has difference engine to run JS codes. Like- Google's Chrome browser has "V8 engine", FireFox has "SpiderMonkey" & Apple's has "JavaScriptCore" for their Safari browser.
The general steps of how a JS engine works: Javascript Source Code > Parser > Abstract Syntax Tree (AST) > Interpreter > Compailer

As we know Javascript is a single threaded interpreted programming language, it reads the code line by line. But some codes (such as: importing data using APIs) takes more time to execute. In that case the browser engine arrange those codes into "stack" and continue executing rest of the codes. From the "stack" the codes are executed within their required time & sent them serially into the "callback queue". After finishing executing main codes line by line the engine run a loop to execute the code from the "callback queue". This loop is called "Event Loop".

The difference between "Local storage" & "Session storage" are:
1. Data Duration: Local Storage(Stores data with no expiration date) & Session Storage(Stores data only for a session)
2. Storage: Local Storage(Upto 10MB) & Session Storage(Upto 5MB)
3. Remove Data: Local Storage(Remove data using JS/clearing the brower cache) & Session Storage(Data removed automatically after closing tab/browser)
